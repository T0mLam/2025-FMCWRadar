# Branch naming convention: 
prefix/#xxx/description-goes-here

## PREFIX
This is to describe which directory you are making changes in.
- **docs:** to do with documentation for the project, e.g dataset, reasearch tutorial.
- **docker:** making changes to docker containers.
- **posture:** changes built on last year's project.
- **scripts:** was used to compile model on the chip.
- **meetings:** meeting notes updates.
- **repo:** anything to do with maintenence of repo .
- **task:** any branching outside of a single directory.

## ISSUENUMBER
*#xxx* where *x* is the issue number relating to the kanban board task this branch is making progress on
## DESCRIPTION
This is a short description to describe the changes being made on the branch at a glance. Any spaces are to be replaced by'-',

*all lowercase*
